## Task Details

### 1. Fix `auto-launch` hardcoded macOS path

**File:** `app/main.dev.js`

**Problem:** Two instances where `AutoLaunch` is constructed with a hardcoded macOS path (`/Applications/Unsplash Wallpapers.app`). On Linux/Windows, `auto-launch` auto-detects the path from `process.execPath` and AppImage/.exe conventions, but the hardcoded mac path breaks it on other platforms.

**Fix:** Make the `path` parameter conditional on `process.platform === 'darwin'`.

### 2. Fix `fix-path` dependency

**File:** `package.json`

**Problem:** `fix-path` is macOS-only. It fixes `$PATH` when launched from Finder.

**Status:** `fix-path` is in `package.json` but **not imported anywhere** in source code. It's a dead dependency. No code change needed; it will be removed from `package.json` as part of cleanup.

### 3. Update `package.json`

Add `win` and `linux` electron-builder config blocks:

- **win**: NSIS installer for both x64 and arm64
- **linux**: AppImage for both x64 and arm64
- **nsis**: Basic NSIS config (oneClick, install dir)

Add new package scripts:
- `package:mac` — macOS ARM64 DMG
- `package:linux` — Linux AppImage (all archs)
- `package:win` — Windows NSIS (all archs)

### 4. Rewrite GitHub Actions

Replace single `macos-latest` job with matrix strategy:

```
build (matrix)
├── macos-arm64      → DMG
├── linux-x64        → AppImage
├── linux-arm64      → AppImage (Docker + QEMU)
├── windows-x64      → NSIS .exe
├── windows-arm64    → NSIS .exe

release (needs build)
└── Collects all artifacts → single GitHub Release
```

**Key differences per runner:**
- macOS: `--mac --arm64`, adhoc signing
- Ubuntu x64: `--linux --x64`, no signing
- Ubuntu ARM64: Docker with QEMU, `--linux --arm64`
- Windows: `--win --x64`, no signing

### 5. Windows ARM64

Added via `windows-11-arm` runner (GitHub public preview, April 2025). Added to the matrix build.
